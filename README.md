# SmartGrid Control Structure (Core 1–Core 5)

## 1. 프로젝트 개요

본 프로젝트는 스마트그리드 환경에서의 에너지 관리 시스템(EMS)과 에너지 저장 장치(ESS) 제어를 출발점으로,  
**계통 안정성이 장비 사양이나 물리적 성능이 아니라 ‘의사결정 구조’에서 발생한다는 가설**을 단계적으로 검증하는 것을 목표로 합니다.

일반적인 스마트그리드 연구는 설비 용량, 출력, 효율 등 하드웨어 성능 비교에 초점을 맞추는 경우가 많습니다.  
그러나 본 프로젝트에서는 동일한 장비 제약 조건하에서도, **제어 논리와 판단 구조에 따라 계통의 안정성 지표가 구조적으로 달라질 수 있음**에 주목하였습니다.

이를 검증하기 위해 Core 1부터 Core 5까지 점진적으로 문제를 확장하였으며,  
부하 구조 분석 → 위험 정의 → ESS 개입 시뮬레이션 → Grid-forming 제어 구조 → 의료 IoT 도메인 확장이라는 흐름으로 구성하였습니다.

본 프로젝트는 다음 사항을 명확히 지양합니다.

- 전력공학 이론에 대한 상세한 수식 설명  
- 장비 스펙 또는 설비 성능의 우열 비교  
- 임상적 효과 또는 의료 성능에 대한 주장  

대신, **제어 구조의 일반성, 의사결정 논리의 재사용 가능성, 도메인 확장성**에 초점을 둡니다.

|---|---|---|
| Core 1 | 부하 구조 분석 | 부하 변동성은 어떤 구조적 패턴을 가지는가? |
| Core 2 | Ramp 기반 위험 정의 | “위험한 변화”는 어떻게 정의되어야 하는가? |
| Core 3 | ESS 개입 시뮬레이션 | ESS는 언제 개입해야 안정성에 기여하는가? |

단일 Core만으로 독립적인 결론을 주장하지 않습니다.

## 4. Core 1 — 부하 구조 분석

Core 1에서는 AMI(Advanced Metering Infrastructure) 데이터를 활용하여 부하의 시간적 구조를 분석하였습니다.

일반적인 부하 분석이 평균값이나 최대값(peak)에 집중하는 것과 달리,  
본 단계에서는 **부하의 절대 크기보다 시간에 따른 변화량(ramp)**에 주목하였습니다.

이를 통해 다음과 같은 사실을 확인하였습니다.

- 부하의 절대값이 크지 않더라도,  
  급격한 변화가 반복될 경우 계통에 누적 부담을 유발할 수 있습니다.
- 안정성 문제는 단일 시점의 값보다,  
  **연속적인 변화 패턴의 구조**에서 발생합니다.

Core 1은 이후 모든 Core에서 사용되는 **ramp 중심 관점의 출발점**을 제공합니다.

---

## 5. Core 2 — Ramp 기반 위험 정의

Core 2에서는 “위험”을 단순히 peak 초과 여부로 정의하는 방식의 한계를 지적하고,  
**Ramp 기반 위험 정의 기준**을 도입하였습니다.

구체적으로, 부하 변화량 |ΔLoad|의 분포를 분석하고,  
상위 분위(예: 95%)를 초과하는 구간을 **위험 ramp**로 정의하였습니다.

이를 통해 다음과 같은 구조적 특성을 확인하였습니다.

- 위험은 드문 이상치(outlier)가 아니라,  
  특정 패턴으로 반복·누적되는 현상입니다.
- 계통 안정성 관점에서 관리해야 할 대상은  
  “큰 값”이 아니라 “급격한 변화”입니다.

Core 2는 이후 ESS 개입 판단 기준의 논리적 근거를 제공합니다.

---

## 6. Core 3 — ESS 개입 시뮬레이션

Core 3에서는 ESS를 도입하여,  
**ESS가 언제 개입해야 안정성 지표가 개선되는지**를 비교 분석하였습니다.

본 단계에서는 ESS 용량, 출력, SOC 제약을 모든 시나리오에서 동일하게 유지하였습니다.  
비교의 초점은 장비 성능이 아니라 **개입 시점과 판단 기준**입니다.

비교 시나리오는 다음과 같습니다.

- Reactive 제어: 임계 초과 발생 이후 사후 대응  
- Proactive 제어: 예측 신호를 활용한 선제 개입  

평가 지표로는 다음을 사용하였습니다.

- 위험 ramp 발생 횟수  
- ramp 누적 정도  
- ESS throughput  

분석 결과, 동일한 ESS 조건에서도 **개입 시점의 차이가 안정성 결과를 구조적으로 변화시킴**을 확인하였습니다.

---

## 7. Core 4 — Grid-forming 제어 구조 (Wind + ESS)

Core 4에서는 재생에너지(풍력) 변동성이 포함된 환경으로 문제를 확장하였습니다.

본 단계의 핵심 목적은,  
풍력 변동성 자체를 “완화했는가”를 평가하는 것이 아니라,  
**제어 구조가 ESS의 개입 시점을 어떻게 재배치하는지**를 비교하는 데 있습니다.

비교 대상은 다음과 같습니다.

- Baseline Reactive (rule-based)  
- Baseline Proactive (forecast-based)  
- Learned Control (저정보 환경에서 proactive 구조를 근사 재현한 모델)

분석 결과, 동일한 풍력–ESS 조건에서도  
예측을 포함한 제어 구조는 ESS 개입을 특정 구간에 집중시키지 않고,  
급격한 부하 변화가 누적되기 이전에 분산적으로 개입함을 확인하였습니다.

이를 통해 다음과 같은 결론을 도출하였습니다.

> 계통 안정성은 장비 사양이 아니라,  
> **제어 논리와 의사결정 구조에서 발생합니다.**

---

## 8. Core 5 — 의료 IoT 확장

Core 5에서는 Core 4에서 정리된 제어 구조를 의료 IoT(페이스메이커) 데이터 구조에 대응시켜,  
**동일한 제어 논리가 산업 도메인을 넘어 확장 가능한지**를 검증하였습니다.

본 단계에서는 스마트그리드와 의료 IoT 간의 구조적 대응 관계를 다음과 같이 설정하였습니다.

| 스마트그리드 | 의료 IoT |
|---|---|
| Load (kW) | Heart Rate |
| ESS SOC | Physiological Reserve |
| Ramp | HR Change |
| Forecast Signal | Next HR |
| EMS Rule | Intervention Logic |

Reactive, Proactive, Context-aware 시나리오를 동일한 제어 규칙 하에서 비교하였으며,  
정보 밀도와 맥락 차이에 따라 개입 시점과 효과가 구조적으로 달라짐을 확인하였습니다.

Core 5는 임상 효과를 주장하지 않으며,  
**제어 구조의 일반성과 다른 CPS/IoT 시스템으로의 확장 가능성**을 제시하는 단계입니다.

## 10. 프로젝트의 역할 정의

본 프로젝트는 특정 설비나 산업에 대한 성능 우위를 주장하지 않습니다.  
대신, **판단 구조와 제어 논리가 어떻게 안정성을 만들어내는지**를 구조적으로 설명하는 것을 목표로 합니다.

이는 스마트그리드뿐 아니라,  
다양한 산업 인프라 및 IoT 시스템으로 확장 가능한 제어 구조 설계의 기초 자료로 활용될 수 있습니다.
